# 🌙 Life Design Map

An interactive 6-month life planning tool synthesising the philosophies of **Tony Robbins**, **Tim Ferriss**, **Brené Brown**, and **Bashar** into a single, beautiful planning interface.

Built with React. Entries auto-save to your browser via `localStorage`.

---

## ✨ Features

- **Vision Map** — overview of all 8 life areas and your 4 mentor guides
- **6-Month Intention Grid** — set monthly intentions across Income, Creative Work, Retreats, Body, Relationships, Travel, Learning, and Inner Life
- **Mentor Lenses** — deep-dive into each philosophy with power questions applied to your specific context
- **Deep Reflection Prompts** — foundational journalling questions before you plan (Fear-Setting, RPM, Wholehearted Living, Highest Excitement)
- **North Star Synthesis** — distil everything into a single 6-month commitment
- **Auto-save** — all entries persist in your browser between sessions
- **Reset** — clear all data and start fresh at any time

---

## 🧭 The Four Philosophies

| Guide | Framework | Core Lens |
|---|---|---|
| 🔥 Tony Robbins | RPM — Result, Purpose, Massive Action | Identity, standards, core needs |
| ⚡ Tim Ferriss | DEAL — Define, Eliminate, Automate, Liberate | Freedom design, fear-setting, 80/20 |
| 💜 Brené Brown | Wholehearted Living — 10 Guideposts | Vulnerability, courage, authenticity |
| ✨ Bashar | Follow Your Highest Excitement | Joy as compass, releasing attachment |

## 🚀 Getting Started

### Prerequisites
- [Node.js](https://nodejs.org/) (v16 or higher)
- npm (comes with Node)

### Installation

```bash
# Clone the repository
git clone https://github.com/YOUR_USERNAME/life-design-map.git
cd life-design-map

# Install dependencies
npm install

# Start the development server
npm start
```

The app will open at `http://localhost:3000`.

## 🌐 Deployment

This project is deployed on [Vercel](https://vercel.com).

To deploy your own version:
1. Fork this repository
2. Go to [vercel.com](https://vercel.com) and connect your GitHub account
3. Import the repository and click **Deploy**

> **Note:** `localStorage` persists entries per device/browser. For cross-device sync, a database integration (e.g. Supabase) would be needed.
